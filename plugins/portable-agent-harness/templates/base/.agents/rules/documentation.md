# 문서화 규칙

## Code-first

- 문서는 실제 코드와 설정을 확인한 뒤 작성한다.
- 파일 경로, 클래스/함수명, CLI 명령, 환경변수, import 경로는 작성 전 실제 repository에서 검증한다.
- 추측으로 API, 옵션, 동작을 문서화하지 않는다.

## 우선순위

불일치가 발견되면 다음 우선순위를 적용한다:

1. 실제 코드와 실행 가능한 설정
2. 공식 프로젝트 문서
3. `AGENTS.md`
4. README와 보조 문서

낮은 우선순위 문서가 높은 우선순위와 다르면 낮은 우선순위 문서를 수정한다.

## 변경 동기화

- 공개 API, CLI, 환경변수, 설정, 디렉터리 구조, 사용자 워크플로가 바뀌면 관련 문서를 함께 수정한다.
- 하위 패키지 또는 앱별 README가 있으면 해당 변경 범위의 README를 확인한다.
- `CHANGELOG.md`가 자동 생성되는 프로젝트에서는 수동 편집하지 않는다.

## 금지

- 코드에 없는 기능을 문서로 약속하지 않는다.
- "나중에", "적절히", "필요시" 같은 모호한 유지보수 지시만 남기지 않는다.
- 특정 티켓이나 사고 이력을 하네스 규칙에 박제하지 않는다. 이력은 ADR, PR 본문, changelog에 둔다.
