# Python 코딩 규칙

## 타입

- 공개 함수와 메서드에는 인자와 반환 타입을 명시한다.
- `Any`는 경계 계층, 테스트 더블, 외부 라이브러리 타입 부재 등 사유가 있을 때만 사용한다.
- `Optional[T]`는 값 부재가 도메인 의미를 가질 때만 사용한다. 빈 문자열, 빈 리스트, sentinel과 혼용하지 않는다.

## 에러 처리

- 라이브러리 또는 애플리케이션 경계에서는 프로젝트 전용 예외 타입을 사용한다.
- silent fallback(`pass`, 무근거 `return None`, 광범위 `except Exception`)을 두지 않는다.
- 사용자 입력, 외부 API, 파일 시스템, 네트워크 경계에서는 실패 원인을 보존한다.

## 구조

- 기존 모듈과 helper를 먼저 검색한 뒤 새 abstraction을 만든다.
- 단일 호출을 감추기 위한 helper를 만들지 않는다. 중복 제거, 의미 부여, 테스트 가능성 중 하나가 실제로 개선될 때만 추출한다.
- 레이어가 있는 프로젝트에서는 안쪽 레이어가 바깥쪽 인프라 구현을 import하지 않는다.

## 금지

- `src/` 또는 애플리케이션 코드에서 사유 없는 `assert`를 사용하지 않는다.
- 사유 없는 `type: ignore`, `pragma: no cover`를 남기지 않는다.
- 런타임 분기를 숨기기 위해 사유 없이 `getattr()`/`hasattr()`/`setattr()`을 사용하지 않는다.
