# PR 리뷰 피드백 반영 워크플로우

## Step 1: 리뷰 댓글 수집

1. `gh pr view $ARGUMENTS --comments`로 PR 리뷰 댓글 확인
2. 각 댓글 분류:

| 분류 | 설명 |
|------|------|
| **필수 수정** | 버그, 타입 에러, 코딩 표준 위반 |
| **개선 제안** | 가독성, 네이밍, 구조 개선 |
| **토론** | 설계 의사결정, 트레이드오프 |
| **무시** | 범위 밖, 의도적 패턴 |

## Step 2: 수정 범위 결정

- 각 댓글에 대해 **최소 변경**으로 해결 가능한지 판단
- 범위를 벗어나는 리팩터링 제안은 별도 이슈로 분리

## Step 3: 수정 적용

1. **타입 에러**: `uv run pyrefly check` 검증
2. **코딩 표준**: `uv run ruff check .` 검증
3. **테스트**: 관련 테스트 실행 및 통과 확인
4. 기존 에러 무시, 내 변경으로 인한 새 에러만 수정

## Step 4: 검증 체크리스트

- [ ] 모든 필수 수정 적용
- [ ] 타입 체크 통과
- [ ] 린트 통과
- [ ] 관련 테스트 통과
- [ ] `Any` 타입, `# type: ignore` 사용 없음

## Step 5: 리뷰어 응답

수정 완료 후 `gh pr comment`로 응답:
- 각 수정 내용 간결하게 설명
- 무시한 제안에 대해 이유 설명
- 별도 이슈로 분리한 항목 링크
