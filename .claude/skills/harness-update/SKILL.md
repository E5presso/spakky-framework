# 하네스 변경 스킬

## 핵심 원칙

| 원칙 | 요약 |
|------|------|
| **환경을 설계하라** | 에이전트 실패 → "환경에 무엇이 빠졌는가?" 질문 |
| **지도를 줘라, 매뉴얼 말고** | 거대 단일 문서는 실패. 분산된 cross-linked docs |
| **단순화 = 성능 향상** | 도구/규칙 제거할수록 에이전트가 빨라지고 안정됨 |
| **Bitter Lesson** | 모델 업그레이드 시 불필요해진 규칙 제거 |
| **제약이 신뢰를 만든다** | 솔루션 공간 확장이 아니라 제약으로 품질 확보 |
| **토큰 = 비용** | 모든 규칙은 토큰 비용 정당화 필요 |

## 컨텍스트 패턴

| 패턴 | 적용 |
|------|------|
| **파일 = 메모리** | 컨텍스트에 모든 것 담지 말고, 파일에 쓰고 필요 시 로드 |
| **Context Rot 대비** | 최소 고신호 토큰만 로드, 나머지는 on-demand |
| **Goldilocks Zone** | 너무 구체적(깨지기 쉬움) ↔ 너무 추상적(추측 유발) 사이 |
| **중복 금지** | 동일 규칙이 두 곳 이상에 기록되지 않도록 |

## 하네스 구조 (Claude Code)

| 파일 | 역할 | 적용 |
|------|------|------|
| `CLAUDE.md` (루트) | 글로벌 지침 | 항상 |
| `*/CLAUDE.md` | 디렉토리 스코프 지침 | 해당 디렉토리 작업 시 |
| `.claude/skills/*/SKILL.md` | 슬래시 커맨드 (스킬) | `/command` 호출 시 |
| `.claude/settings.json` | 훅, 권한 (공유) | 항상 |
| `.claude/settings.local.json` | 로컬 전용 설정 (gitignore) | 항상 |

## 변경 원칙

1. **최소 범위**: 글로벌 → `CLAUDE.md` | 디렉토리 → 하위 `CLAUDE.md` | 워크플로우 → `skills/`
2. **중복 금지**: 한 곳에만 기록
3. **쓰기 보호**: 하네스 변경 전 사용자 승인 필요

## 워크플로우

1. **위치 결정**: 규칙 적용 상황 분석 → 적합한 파일 선택
2. **제안**: 전체 변경 내용 마크다운 출력 → 승인 대기
3. **적용**: 승인 후 파일 수정 → 정합성 확인
