# 에이전트 헌장

이 프로젝트에서 에이전트가 따르는 **최상위 의사결정 위계**다. 다른 모든 규칙은 이 헌장에 종속되며, 헌장과 충돌하면 헌장을 따른다.

## 1. 의사결정 위계

```
정책(Policy)  →  비즈니스(Business)  →  코드(Code)
```

- **정책**: 사용자가 명시한 작업 범위, 마일스톤 의도, 하네스 규칙. 가장 강한 권위.
- **비즈니스**: 도메인 모델, 패키지 책임, 공개 API 계약. 정책에 종속된다.
- **코드**: 구현 디테일. 비즈니스 결정에 종속된다.

**상위 결정 없이 하위로 내려가지 않는다.** 코드 구조를 먼저 잡고 비즈니스를 끼워 맞추거나, 비즈니스 결정을 정책 검토 없이 확정하지 않는다.

## 2. 3-축 정렬 (기술 진입 사전 검증)

티켓·이슈·기능 작업에 코드를 한 줄도 쓰기 전에, 다음 3축을 2-3줄로 명확히 진술한다. 셋 중 하나라도 모호하면 **기술 진입 금지**, 사용자에게 확인한다.

| 축 | 질문 | 모호 시 |
|----|------|--------|
| **마일스톤 의도** | 이 작업이 어떤 상위 목적에 기여하는가? | 마일스톤/에픽 본문 재독 후 재진술 |
| **티켓 위치** | 이 티켓이 마일스톤의 어느 단계에 위치하는가? blocker는 해소되었는가? | 의존 그래프에서 위치 확인, blocker 확인 |
| **도메인 사전** | 등장 어휘(Aggregate·Port·UseCase 등)가 현재 도메인 모델에 존재하는가? | `core/spakky-domain/`·관련 패키지에서 검증, 없으면 신설 정당성 확인 |

이 검증은 형식이 아니다. 구식 전제 위에 정교한 코드를 쌓는 것을 막는 안전장치다.

## 3. 7차원 품질 바

코드 리뷰·자가 검토 시 "동작하는가"만이 아니라 다음 7차원으로 평가한다.

1. **간결성**: 같은 결과를 더 적은 줄·더 적은 추상화로 달성할 수 있는가?
2. **자기 설명성**: 식별자만 보고 의도를 알 수 있는가? 주석으로 무엇(WHAT)을 설명하지 않는가?
3. **최신 문법**: Python 3.11+ 표준 문법을 쓰는가? `List[T]`/`Optional[T]`/`from __future__ import annotations` 같은 레거시는 금지.
4. **성능과 가독성 동시 보전**: N+1, 불필요한 직렬화, O(N²) 등 명백한 안티패턴은 작성 시점에 회피한다. 미시 최적화는 hot path 식별 후만.
5. **모듈성**: SOLID 5원칙(SRP·OCP·LSP·ISP·DIP) 검증. 한 클래스가 한 책임만 갖는가?
6. **레이어·의존 방향**: `monorepo.md`·`domain.md`의 단방향 의존 규칙 위반 없음.
7. **대원칙 준수**: 헌장·`behavioral-guidelines.md`의 모든 항목과 충돌 없음.

7차원 중 하나라도 위반하면 PR 머지 금지 또는 후속 티켓 분기.

## 4. 외부 게이트 (자기확증 편향 차단)

자기 판정만으로는 편향이 누적된다. 다음 게이트는 **반드시 외부**(서브에이전트·사용자·자동화 스크립트)가 통과시켜야 한다.

| 게이트 | 외부 주체 | 차단 조건 |
|--------|---------|---------|
| 코드 리뷰 | `/review-code` 서브에이전트 | Critical 1개 이상 → 커밋 차단 |
| 회고 | `/retro` 서브에이전트 | (보고만, 차단 없음) |
| 하네스 평가 | `/review-harness` 서브에이전트 | 선언-실행 단절 발견 시 |
| 사용자 의도 정렬 | 사용자 회신 | 3-축 모호 시 사용자 답변 필수 |
| 사양 감사 | `/audit-spec` (해당 시) | Critical 발견 시 |

**자기 판정으로 게이트를 우회하는 것은 헌장 위반이다.**

## 5. 모호함 처리 (단계별 강도 차등)

| 단계 | 강도 | 행동 |
|------|------|------|
| 정책 단계 | 편집증적 | 1mm 모호함도 사용자에게 질의 |
| 비즈니스 단계 | 신중 | 도메인 사전·기존 코드 우선 참조, 없으면 질의 |
| 구현 단계 | 자율 | 명확한 결정 기준이 있으면 자율 진행, 트레이드오프만 보고 |

상위 단계의 모호함을 하위 단계에서 "구현으로 채우지 않는다." 정책 모호 → 코드 작성으로 추측 채우기는 헌장 위반.

## 6. 토큰 = 비용

하네스·메모리·문서·코드 모두 토큰 비용을 발생시킨다.

- 같은 차단력을 더 적은 토큰으로 얻을 수 있으면 그쪽이 default.
- 규칙·메모리 추가 시 `harness-writing.md`의 5-test 게이트를 통과해야 한다.
- "혹시 모르니까"로 추가하지 않는다. 명백한 비용 대비 가치가 있을 때만.
